# Checkpoint 5 - Parte 2

🧪Exercício – Autenticação com React Hook Form e json-server

1 - Crie um projeto VITE+REACT+TYPESCRIPT com o nome access-control-cp , configure o json-server no projeto e crie o endpoint usuarios com os campos: id, nome, nomeUsuario, email.(5)

2- Crie a rota /login como página inicial da aplicação.(5)

3 - Crie a rota /cadastro, acessível a partir de um link ou botão em /login.(5)

4 - Na página Login, crie um formulário com os campos nomeUsuario e email, validando-os com React Hook Form.(5)

5 - Na página Cadastro, crie um formulário com os campos nome, nomeUsuario e email, validando-os com React Hook Form.(5)

6 - Implemente mensagens de erro personalizadas em ambos os formulários, exibindo feedback abaixo dos inputs, tratando todos os campos com HookForms.(5)

7 - Implemente a lógica de cadastro: antes de cadastrar um novo usuário, verifique duplicidade de nomeUsuario e email no json-server.(5)

8 - Implemente a lógica de login: valide o usuário consultando o endpoint usuarios e simule autenticação com localStorage/sessionStorage.(5)

9 - Se o usuário for validado, apresente o nome  e o email  do usuário em todas as páginas, procure a melhor estratégia para realizar este processo.(5)

10 - Versione o projeto com mínimo de 30 commits bem descritos e entregue o repositório atualizado no GitHub e com rótulos específicos nos commits.(5)
Ex: Se a atualização tem a ver com caracteristicas gerais coloque o rótulo feat:, se for resolução de problemas issue:, novos requests enhancement:, para dúvidas sobre estes rótulos veja a apostila de Git/Github disponível na área do aluno.
